(.) - Any character, alphanumeric or others such as #, -, % and so on,

(except \n newline)

\w – any alphanumeric character

will match any character a-z, upper case A_Z and number 0-9

For example – the regular expression ca\w will match “car”, “cab”, “cat” “can”, “ cap” and so on.

Comments:

If it was not chosen as a standalone word with word boundaries (i.e., ‘\bca\w\b’), which will make this a 3 latter word only, starting with “ca..”,

other strings will be matched like “American”, “theoretically”, “categories”

\d – digit

will capture any digit 0-9

For example – if you want to capture all ages between 10-19 years you may choose 1\d – then followed by years of months with an optional white space. This will capture 10,11,12,...19

Comments:

If it was not chosen as a standalone word with word boundaries (i.e., ‘\b1\d\b’) which will

make this a 2 digit/character “word” only, starting with “1..”, other strings will be matched like “2015”, “6674451334”….

\s will capture any white space. Including space, Tab etc.

For example – if you are trying to capture some demographic features like “Age: 19 years”. If this is part of your EMR template, the words may be separated by a space or by a Tab or a new line. The feature \s will help you capture all those strings. You will write is as Age:\s19\syears

Comments:

The words “Age:” “19” and “years” may be separated by more than one space or tab. You may want to consider adding “+”

= which means ‘one or more times’. This way you capture the expression with as many white spaces there are. The expression will then be – Age\s+19\s+years

Sets – [a-z], [0-9]

You will use these when you want to capture something within a range. Typically it will not be [0-9] because you may choose to use \d instead. But it can be a shorter range like [0-5]. You may choose [a-z] over \w when you want to capture a letter, but not a number.

For example – if you want to capture patients with hypothermia and a temperature ranging from 30-35, you can use 3[0-5]. With that you will capture “30”..”31”…->..”35”

Note the square brackets. They suggest you pick one from the range offered. Had you used 3(0-5) you would match the literal content of the brackets, i.e., “30-5”

? – this symbol following a character or a contained string suggests that that character or string is optional, i.e., it can appear “0” or more times.

For example – the dosing using 25mh may appear as “25mg” or using a white space “25 mg”. To capture both I want the white space to appear.. you got it ! “0’ or more times, or in other words, be optional

The expression I can use is “25\s?mg” which will capture both 25mg and 25 mg

Comments:

What happens when I want to use a question mark? In regular expressions there are some characters that we call metacharacters – meaning that while they have a literal meaning (like a question mark), they are used as a symbol in regular expression. For that reason, if you want to override the metacharacter properties, you need to use a “\” sign prior to using the character.

So the expression PCP\? Will capture “PCP?”

While as the expression PCP? – will be using the “?” as a metacharacter suggesting that the last letter “P” is optional, and therefore capture “PCP” and “PC”

_______________________________________________________________

+ - this symbol suggests a repetition as well. Only it refers to having the previous character or contained string appears one time or more.

For example the expression 31+ will capture “31… 311… 3111 …” where the last character ‘1’ can appear 1-∞ times

Comments:

The “+” sign is a metacharacter. If you need to use the + sign in its original literal meaning, you need to use “\” as in \+

_______________________________________________________________

*

This symbol is basically a combination of the metacharaters “?’ and “+” suggesting the preceding character or contained string is both optional or can be repeated many times. In other words it can appear 0-∞ time.

For example- the expression address* captures addres, address, addresss, addressss….

Comments:

The * sign is a metacharacter, if you need to use it as an asterisk you need to write it with a preceding “\”. As in “\*” – so the expression Address\* will capture Address*

[0-9]

Any character that is a digit

Either or

Can contain arbitrarily complex regex's on either side of the pipe

For example: (Hello|Hola|Bonjour|Hallo|Ciao)

Range - Matching Numeric Ranges

The regex [0-9] matches single-digit numbers 0 to 9

[1-9][0-9] matches double-digit numbers 10 to 99

Since regular expressions deal with text rather than with numbers, matching a number in a given range takes a little extra care. You can't just write [0-255] to match a number between 0 and 255. Though a valid regex, it matches something entirely different. [0-255] is a character class with three elements: the character range 0-2, the character 5 and the character 5 (again). This character class matches a single digit 0, 1, 2 or 5, just like [0125].
